WSF: A One Hour Compo Revelation

Through the use of MiniWSF and MeltWSF, one hour compo votepacks can become amazingly smaller! How? Read this following guide:

  • What is WSF/WTF?

WSF (Whelchel Sound Format) and WTF (Whelchel Tune Format) are two different names for the same thing. The WTF extension was adopted after the conflict with Windoze Scripting Format was identified. WSF is a simply a means of compressing a group of modules with duplicate or simply modified samples (reversed, appended, cut, etc).

  • What does this mean for an OHC?

Well, for those who don't know, One Hour Compos (OHCs) usually involve Sample Packs. The entriants in an OHC are limited to using these samples. This means that these samples will be used many times throughout many songs when it comes to voting time. There are many compos based on OHC format as well, including Bar Wars. WSF will take these modules, and remove the samples from all of them and store them in a WSP file. This acts as the 'Sound Bank' for our tracks. Put Simply: Voters will be downloading each sample once, instead of once for every track.

Getting Started

So you probably want to get started using WSF. There are a number of ways:

  • OHC Hosts on a Windows system
    OHC Hosts on Windows simply need to download MiniWSF from the download page. They run the executable, and will select the modules that were entered. A self extracting windows executable will then be built (a 68kb basefile!) from these modules, and you'll see a signifigant size decrease. YAY. But what about the voters that aren't on windows?! A good question. MeltWSF source should be included in the votepack for *nix users. (MeltWSF is gcc compliant) The source is found on the downloads page in it's own small package (opposed to the full WSF Source).
  • OHC Voters not on Windows
    OHC Voters should obtain the MeltWSF source (MeltWSF is gcc compliant). MeltWSF can take a MiniWSF windows executable (without actually executing it) and extract the music. Very simple.
  • OHC Hosts NOT using Windows
    OHC Hosts that aren't on Windows should obtain the MeltWSF source and compile it. (MeltWSF is gcc compliant). To build a votepack (that requires MeltWSF to extract) they run MeltWSF like so:
    MeltWSF build myvotepackfile.mwp "my ohc title" "my message" file1 file2 file3

  • OHC Voters voting with a nonexecutable pack
    1. Obtain MeltWSF (either the source, or the windows binary) from the downloads section.
    2. Run make -f meltmake to build it (if you have the source)
    3. Run MeltWSF like so: meltwsf mypack.mwp
